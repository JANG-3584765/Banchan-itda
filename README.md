# 🥗 동네 반찬가게 리뷰 및 큐레이션 플랫폼: 반찬잇다

> **"오늘 뭐 먹지?" 고민하는 자취생부터 직장인까지, 우리 동네 숨은 '집밥 맛집'을 한눈에.**

배달 음식의 자극적인 맛과 공장형 밀키트의 한계를 넘어, 소상공인의 정성이 담긴 반찬가게 정보를 통합하고 리뷰하는 위치 기반 서비스입니다.

---

## 💡 Project Background (기획 배경)

### 1. 현대인의 '식(食)' 문제 해결
- 배달 음식 및 고칼로리 외식에 대한 피로도 증가.
- 건강한 집밥을 원하지만, 직접 요리하기 어려운 1인 가구 및 맞벌이 가정의 증가.
- 공장형 밀키트가 채워주지 못하는 '손맛'과 '신선함'에 대한 수요 존재.

### 2. 정보의 파편화 및 격차 해소
- **문제점:** 현재 반찬가게 정보는 네이버 카페, 당근마켓, 지역 커뮤니티 등에 흩어져 있거나 아예 등록되지 않은 경우가 많음.
- **니즈:** 이사 온 지 얼마 안 된 거주자나 정보 접근성이 낮은 사용자는 검증된 동네 맛집을 찾기 어려움.
- **목표:** 흩어진 정보를 한데 모아 '반찬 특화' 검색 및 리뷰 환경 제공.

## ✨ Core Features & Differentiation (차별점)

| 구분 | 기존 플랫폼 (네이버/당근) | **본 프로젝트의 차별점** |
| :--- | :--- | :--- |

| **콘텐츠** | 정적인 매장 정보 | **'오늘의 라인업' 구독 및 알림** (예정) |

## 🛠 Tech Stack & Roadmap (기술 스택 및 단계별 확장 계획)

본 프로젝트는 기술적 숙련도를 점진적으로 높여가는 과정을 기록하는 것을 목표로 합니다.

### 🏁 Phase 1: 기반 구축 (Current)
- **Markup & Styling:** HTML5, CSS3 (BEM 방법론 활용)
- **Language:** JavaScript (Vanilla JS)
- **Focus:** 시맨틱 마크업, 반응형 레이아웃, 기초 UI 컴포넌트 설계

### 🚀 Phase 2: React 전환 및 고도화
- **Framework:** React, Next.js
- **Styling:** Emotion (CSS-in-JS)
- **State Management:** - **Client:** `Jotai` (장바구니, 필터 상태 등 전역 상태 관리)
  - **Server:** `Tanstack Query` (리뷰 데이터 패칭 및 캐싱 최적화)

### ⚙️ Phase 3: 인프라 및 최적화
- **Map API:** 카카오맵 / 네이버 지도 API 연동 (위치 기반 서비스)
- **Performance:** Next.js Image Optimization (Lazy Loading), 이미지 WebP 변환
- **CI/CD:** GitHub Actions, Vercel/AWS 배포
- **Package Manager:** Yarn Berry / PNPM
